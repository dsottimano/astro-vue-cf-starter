# Photos & media

[← Back to wiki](index.md)

Images make listings and posts shine. Here's how uploading works.

## How to upload

- **Listings:** in the listing form, use the **Photos** (and **Floorplans**)
  sections — click the file picker and choose one or more images.
- **Blog posts:** use the **Cover image** uploader in the post form.

You can select **several files at once**. Each uploaded image appears in the list
with a short code (its storage key) — that's normal, you don't need to do anything
with it. Use the ✕ next to an image to remove it from the listing.

## Formats & size limits

| | Allowed |
|---|---|
| **Photos / cover images** | JPEG, PNG, WebP, AVIF |
| **Floorplans** | JPEG, PNG, WebP, AVIF, **and PDF** |
| **Maximum size** | 15 MB per file |

If an upload is rejected, it's almost always because the file is **too large** or
an **unsupported type** (e.g. a HEIC straight from an iPhone, or a `.docx`).
Convert it to JPEG/PNG and try again. On an iPhone you can change
**Settings → Camera → Formats → Most Compatible** to get JPEGs.

## Good to know

- **Photos go live immediately.** Unlike text changes, uploaded images are
  available right away. But the listing/post that *displays* them still needs the
  usual minute-or-two rebuild to show up — see
  [How publishing works](publishing.md).
- **Order matters.** The first photo is typically used as the main/cover image.
- **Re-using images:** each upload is stored separately. There's no shared media
  library yet — upload the image where you need it.
- **Big photos slow the page.** Aim for web-sized images (roughly 1600–2400px
  wide) rather than full 50-megapixel originals.
