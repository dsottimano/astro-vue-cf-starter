# Categories

[← Back to wiki](index.md)

Categories are the folders that organize **blog posts** (for example "News",
"Market Updates", "Guides"). Open **Categories** 🏷 in the left menu.

## How categories work

- Each category has a **slug** (its web-address piece, e.g. `market-updates` →
  `/en/blog/category/market-updates`).
- Each category has a **name per language** — e.g. English "News" / Spanish
  "Noticias" — so the same category shows correctly in both languages.
- A category can optionally have a **parent**, letting you nest one level
  (e.g. "Guides" → "Buyer Guides").

## Add a category

1. Click **Categories → Add** (or the add control).
2. Enter the **slug** (lowercase, dashes instead of spaces).
3. Enter the **name for each language**.
4. (Optional) Pick a **parent** category.
5. **Save.**

## Use a category on a post

Categories you create here show up as tick-boxes in the
[blog post](blog-posts.md) form. A post can belong to more than one.

## Rename or remove

- **Renaming** the display name is safe — edit it and Save.
- **Changing the slug** changes the category's web address (and any links to it),
  so avoid it once the category is in use.
- **Deleting** a category removes the folder; posts that referenced it simply lose
  that tag.

As with everything else, changes appear after the usual short rebuild
([How publishing works](publishing.md)).
